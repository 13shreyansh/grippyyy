Quick Start

Get started with the Grippy API in 3 steps:

1. Create an account and get an API key

# Register
curl -X POST /api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"you@co.com","username":"dev","password":"secret123"}'

# Login to get JWT token
curl -X POST /api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email_or_username":"dev","password":"secret123"}'
# Returns: {"token": "eyJ..."}

# Create an API key
curl -X POST /api/b2b/keys \
  -H "Authorization: Bearer eyJ..." \
  -H "Content-Type: application/json" \
  -d '{"name":"my_app","tier":"free"}'
# Returns: {"key": "grp_live_abc123...", "key_id": "..."}

2. Submit a complaint

curl -X POST /api/b2b/complaint \
  -H "X-API-Key: grp_live_abc123..." \
  -H "Content-Type: application/json" \
  -d '{
    "company": "IndiGo Airlines",
    "issue": "Flight 6E-2341 delayed 5 hours, no food provided",
    "user_name": "Rahul Sharma",
    "user_email": "rahul@example.com",
    "action": "auto"
  }'
# Returns: {"complaint_id": "...", "task_id": "..."}

3. Check the result

curl /api/v3/tasks/TASK_ID \
  -H "X-API-Key: grp_live_abc123..."
# Returns: {"status": "completed", "result": {...}}

Authentication

All B2B API endpoints require an X-API-Key header. API keys are created through the /api/b2b/keys endpoint using JWT authentication.

B2B Endpoints

POST /api/b2b/complaint
Submit a complaint for automated processing. Grippy will find the right form, draft an email, and optionally send it.
ParameterTypeDescription
company requiredstringCompany name to complain against
issue requiredstringDetailed description of the complaint
user_namestringComplainant's full name
user_emailstringComplainant's email address
user_phonestringComplainant's phone number
actionstringOne of: auto, email, form, escalate (default: auto)
GET /api/b2b/complaint/{complaint_id}
Get the status and details of a submitted complaint.
POST /api/b2b/keys
Create a new API key. Requires JWT auth via Authorization header.
GET /api/b2b/keys
List all API keys for the authenticated user.
POST /api/b2b/keys/revoke
Revoke an API key.
GET /api/b2b/usage
Get API usage statistics for your key.

Other Useful Endpoints

GET /api/v3/search?q={query}
Search the web for complaint forms and company info.
POST /api/v3/tasks/submit
Submit a background form-fill task. Returns a task_id for polling.
GET /api/v3/tasks/{task_id}
Check the status and result of a background task.
GET /api/health
System health check — browser pool, email, scheduler, search, task queue.
